Webtopie build system —HTML5 Test Page

This is a test page filled with common HTML elements to be used to provide visual feedback whilst building CSS systems and frameworks.

With a simple icon system , a sass starter pack and some JS utilities

When the js interacts with the DOM, we use attributes (classical, aria & custom data) to interact & transmit information. We do not rely on classes to avoid breaking changes problems (in case the style of an element changes but the functionality stays the same.)

We use most of the CUBE CSS methodology as a base for the CSS. It is simple, loose, aligned with the spirit of CSS and can be combined with other methodology/naming conventions (like BEM) for more information, see the CUBE CSS website.

classes are separated when declared in the HTML using "|" to differentiate between layout, utility & block type classes. like this : class="button--primary | is-button-color-primary".the "|" character is not used as an independant class in our stylesheet so the browser just ignores it. button--primary is a block class, is-button-color-primary is an utility. Ideally we want this order : class="block | layout | utility"

We try to keep everything as consistent as possible.

Paragraphs

A paragraph (from the Greek paragraphos, “to write beside” or “written beside”) is a self-contained unit of a discourse in writing dealing with a particular point or idea. A paragraph consists of one or more sentences. Though not required by the syntax of any language, paragraphs are usually an expected part of formal writing, used to organize longer prose.

Blockquotes

A block quotation (also known as a long quotation or extract) is a quotation in a written document, that is set off from the main text as a paragraph, or block of text.

It is typically distinguished visually using indentation and a different typeface or smaller size quotation. It may or may not include a citation, usually placed at the bottom.
